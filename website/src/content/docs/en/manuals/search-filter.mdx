The search bar lets you instantly filter the media grid to find specific files within the currently open folder. It is located at the top of the media grid area.

## Using the Search Bar

| Key            | Action                                                               |
| :------------- | :------------------------------------------------------------------- |
| <Kbd>I</Kbd>   | Focus the search input and start typing                              |
| <Kbd>Esc</Kbd> | Unfocus the search bar, returning keyboard control to the media grid |
| **Click**      | Click the input field to focus it                                    |

As you type, the grid updates in real time to show only files whose names match your query. The search is case-insensitive.

## What Gets Searched

Search queries are matched against **filenames only** (including the file extension). For example:

| Query      | Matches                                   |
| :--------- | :---------------------------------------- |
| `vacation` | `vacation_2024.jpg`, `beach-vacation.png` |
| `.mp4`     | All MP4 video files                       |
| `.flac`    | All FLAC audio files                      |
| `photo`    | `photo1.jpg`, `Screenshot_photo.png`      |

Filtering by extension (e.g., typing `.mp4`) is a quick way to isolate all files of a specific format.

## What Gets Excluded

Search does **not** inspect internal file metadata (EXIF tags, audio artist/album names, video container metadata). It is strictly a filename filter. If you need to find files by metadata, use the [Metadata Panel](/manuals/metadata-panel/) to inspect individual files.

## Search State

- When a search is active, the filter text is displayed in the search bar.
- Pressing <Kbd>Esc</Kbd> unfocuses the search bar and returns keyboard control to the grid.
- Clicking the clear button (×) or manually deleting the text resets the filter and shows all files again.
- Changing the active folder also clears the search query.
