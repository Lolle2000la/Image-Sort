このページでは、詳細な設定項目と、フォルダ操作およびファイル削除プロファイルの設計仕様について説明します。

## フォルダの挙動設定

Media Sortでは、開いたディレクトリに対するアプリの挙動を調整できます。

### 1. 最後に開いたフォルダを再開する
デフォルトでは、最後に開いていたフォルダ（ワークスペース）を記憶し、次回起動時に自動的に再開します。
- **設定キー:** `general.reopen_last_opened_folder` (真偽値)
- **挙動:** `true` の場合、アプリは `general.last_opened_folder` にパスを保存し、起動時に自動的に読み込みます。起動するたびに必ずフォルダ選択ダイアログを表示させたい場合は `false` に設定します。

### 2. フォルダツリーの幅
- **設定キー:** `general.folder_tree_width` (数値)
- **挙動:** 左側のフォルダナビゲーションツリーのデフォルト幅をピクセル単位で定義します。

## ファイルの削除とシステムのゴミ箱

Media Sortの設計における重要な基本原則は、**「安心の整理（非破壊の削除）」**です。これにより、即座の**取り消し（Undo）/ やり直し（Redo）**がサポートされます。

### 削除の仕組み

ファイル削除（`下矢印キー`）を実行した際、ファイルは直接物理的に削除されるのではなく、お使いのオペレーティングシステムのゴミ箱に移動されます。

- **Windows:** ネイティブのシェルAPIを使用して「ごみ箱」に移動します。
- **macOS:** Cocoaの `NSFileManager` APIを使用して「ゴミ箱」に移動します。
- **Linux:** Freedesktop.orgのTrash仕様に従って、ユーザーのゴミ箱に移動します。

### 取り消し（Undo）と復元

ファイルがシステムのゴミ箱に退避されているため、取り消しキー（`Q`）を押すことで、元のパスにファイルをプログラムで完全に復元できます。

1. **状態の保持:** Media Sortは元のファイルパス情報を含む参照ハンドル（`TrashRestoreHandle`）を保持します。
2. **メモリの解放とフラッシュ:** アプリケーションが実行されている間、これらのハンドルは元に戻す（Undo）スタックに保持されます。アプリケーションを終了すると、Undoスタックがクリアされてハンドルが破棄されます。破棄される際、削除アクションが「フラッシュ（確定）」され、アプリ内からのUndoは行えなくなりますが、OSのゴミ箱にはそのまま残るため、必要に応じてOS側から手動で戻すことも可能です。
