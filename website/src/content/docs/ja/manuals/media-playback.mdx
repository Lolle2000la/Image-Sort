Media Sortには、完全な再生操作機能を備えた内蔵の音声・動画プレーヤーが搭載されています。音声と動画は異なるバックエンドシステムで処理されます。

## 画面上の操作

音声と動画のプレビューペインは、共通の画面上操作コントロールを共有しています。

| 操作                | 説明                                         |
| :------------------ | :------------------------------------------- |
| **再生 / 一時停止** | 再生の開始または一時停止                     |
| **停止**            | 再生を停止し、位置を先頭にリセット           |
| **シークバー**      | クリックまたはドラッグで特定の位置にジャンプ |
| **音量スライダー**  | 再生音量の調整                               |
| **ミュート切替**    | 音声のミュート/ミュート解除                  |

## 再生のキーボードショートカット

これらのショートカットは、再生可能なメディアファイルが選択されているときに有効です。これらは**ハードコード**されており、設定ダイアログで再マッピングすることはできません。

| キー                                                                     | アクション                                      |
| :----------------------------------------------------------------------- | :---------------------------------------------- |
| <Kbd>Space</Kbd>                                                         | 再生 / 一時停止の切り替え                       |
| <Kbd>MediaPlayPause</Kbd> / <Kbd>MediaPlay</Kbd> / <Kbd>MediaPause</Kbd> | 再生 / 一時停止の切り替え（マルチメディアキー） |
| <Kbd>MediaStop</Kbd>                                                     | 再生停止                                        |
| <Kbd>MediaTrackNext</Kbd>                                                | 次のメディアファイルを選択                      |
| <Kbd>MediaTrackPrevious</Kbd>                                            | 前のメディアファイルを選択                      |
| <Kbd>AudioVolumeUp</Kbd>                                                 | 音量を5%上げる                                  |
| <Kbd>AudioVolumeDown</Kbd>                                               | 音量を5%下げる                                  |
| <Kbd>AudioVolumeMute</Kbd>                                               | ミュート切り替え                                |

## 動画再生

動画再生は**libmpv**（FFmpeg経由）によって動作します。動画フレームは、ゼロコピーのGPUアクセラレーション表示のためにカスタムwgpuシェーダーを使用してレンダリングされます。

### システム要件

- **libmpv**がシステムにインストールされている必要があります。Linuxでは`libmpv-dev`または`libmpv1`パッケージをインストールしてください。macOSとWindowsでは、mpvはアプリケーションに同梱されています。
- 動画レンダリングには**Vulkan対応GPU**と最新のドライバが必要です。
- libmpvが見つからない場合、アプリ内での動画再生は無効になりますが、動画ファイルは引き続き検出され、ブラウズできます。画像と音声は通常通り動作します。動画のサムネイルもmpvが必要なため、代わりにプレースホルダーが表示されます。

### 対応動画フォーマット

対応動画フォーマットの完全なリストについては、[メディア対応状況](/manuals/media-capabilities/)のページをご覧ください。フォーマット対応は、インストールされているlibmpvのバージョンとその基盤となるFFmpegコーデックに依存します。

### GIF再生

GIFファイルはMedia Sortでは動画として分類されます。プレビューとグリッドでGIFをアニメーションさせるかどうかを制御するには、設定ダイアログの**「アニメーションGIFを再生」**設定を使用します（`config.toml`では`animate_gifs`としても利用可能）。無効にすると、GIFは最初のフレームのみを静止画像として表示します。

## 音声再生

音声は**Symphonia**（全コーデック）でデコードされ、**Rodio**を通じて出力されます。アプリケーションに同梱されているもの以外の外部依存関係は不要です。

### 対応音声フォーマット

対応音声フォーマットの完全なリストについては、[メディア対応状況](/manuals/media-capabilities/)のページをご覧ください。

### アルバムアート

埋め込みカバーアートを含む音声ファイル（MP3、FLAC、M4A、OGGで一般的）は、プレビューペインに汎用の音楽アイコンの代わりにカバー画像を表示します。

## フォールバック: システムプレーヤーで開く

Media Sortがファイルをデコードできない場合（例: mpvが不足している、またはサポートされていない画像形式）、プレビューペインに**「システムプレーヤーで開く」**または**「外部で開く」**ボタンが表示されます。クリックすると、システムのデフォルトアプリケーションでファイルが起動されます:

- **Linux:** `xdg-open`
- **macOS:** `open`
- **Windows:** `cmd /C start`
