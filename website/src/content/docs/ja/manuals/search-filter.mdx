検索バーを使うと、現在開いているフォルダ内の特定のファイルを見つけるためにメディアグリッドを即座にフィルタリングできます。検索バーはメディアグリッド領域の上部にあります。

## 検索バーの使い方

| キー | アクション |
| :--- | :--- |
| <Kbd>I</Kbd> | 検索入力にフォーカスして入力を開始 |
| <Kbd>Esc</Kbd> | 検索バーのフォーカスを外し、キーボード操作をメディアグリッドに戻す |
| **クリック** | 入力フィールドをクリックしてフォーカス |

入力すると、グリッドはリアルタイムで更新され、クエリに一致するファイル名のファイルのみが表示されます。検索は大文字と小文字を区別しません。

## 検索対象

検索クエリは**ファイル名のみ**（ファイル拡張子を含む）と照合されます。例:

| クエリ | 一致例 |
| :--- | :--- |
| `vacation` | `vacation_2024.jpg`、`beach-vacation.png` |
| `.mp4` | すべてのMP4動画ファイル |
| `.flac` | すべてのFLAC音声ファイル |
| `photo` | `photo1.jpg`、`Screenshot_photo.png` |

拡張子でフィルタリングする（例: `.mp4`と入力する）ことは、特定の形式のファイルをすべて素早く抽出する便利な方法です。

## 検索対象外

検索は内部ファイルメタデータ（EXIFタグ、音声のアーティスト名・アルバム名、動画コンテナメタデータ）を**検査しません**。厳密にファイル名フィルターです。メタデータでファイルを探す必要がある場合は、[メタデータパネル](/manuals/metadata-panel/)を使用して個別のファイルを検査してください。

## 検索状態

- 検索がアクティブな場合、フィルターテキストが検索バーに表示されます。
- <Kbd>Esc</Kbd>を押すと検索バーのフォーカスが外れ、キーボード操作がグリッドに戻ります。
- クリアボタン（×）をクリックするか、テキストを手動で削除するとフィルターがリセットされ、再びすべてのファイルが表示されます。
- アクティブなフォルダを変更した場合も、検索クエリはクリアされます。
