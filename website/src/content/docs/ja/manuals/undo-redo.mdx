Media Sortは、移動、コピー、名前変更、削除といったすべての仕分け操作を履歴スタックに記録し、ミスを即座に取り消したり、気が変わった操作をやり直したりすることができます。

## キーボードショートカット

| キー         | アクション               |
| :----------- | :----------------------- |
| <Kbd>Q</Kbd> | 最後の操作を取り消し     |
| <Kbd>E</Kbd> | 取り消した操作をやり直し |

## 取り消し可能な操作

以下の操作は取り消し可能です。

- **移動**: ファイルが元の場所に戻されます
- **コピー**: コピー先のファイルが削除されます
- **名前変更**: ファイル名が元の名前に戻されます
- **削除**: ファイルがシステムのゴミ箱から元の場所に復元されます

## 削除の仕組み

ファイルを削除する（<Kbd>Down</Kbd>を押す）と、ディスクから**完全に削除されるわけではありません**。代わりに、OSネイティブのゴミ箱機能に移動されます。

- **Windows:** Windows ゴミ箱（ネイティブシェルAPI経由）
- **macOS:** ゴミ箱フォルダ（Cocoa `NSFileManager`経由）
- **Linux:** デスクトップゴミ箱（Freedesktop.org trash仕様）

ファイルはシステムのゴミ箱に残っているため、削除の取り消しはプログラムによってファイルを正確に元のパスに復元します。

## 履歴のライフサイクル

- **アプリの実行中**は、すべての操作が取り消し/やり直しスタックに保持され、いつでも元に戻せます。履歴の深さに固定の制限はありません。
- **アプリを終了する**と、履歴スタックはクリアされます。取り消しスタックにあった削除操作は確定され、Media Sortからそれらを取り消すことはできなくなります。
- **終了後**も、削除されたファイルはシステムのゴミ箱に存在し、OSのファイルマネージャーを使って手動で復元できます。

## 制限事項

- Media Sortの外部で後から変更または移動されたファイルが関与する操作は、正しく取り消せない場合があります。その場合、操作はスキップされ通知が表示されます。
- ファイルシステム監視イベント（外部プログラムによるファイルの追加、変更、削除）は取り消し履歴に記録されません。
