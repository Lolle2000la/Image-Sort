Media Sortは、幅広いメディアフォーマットをサポートするフル機能のメディア整理ツールです。ファイル構造から直接詳細なメタデータを抽出し、インスペクタでの確認や検索バーによる絞り込みを可能にします。

## サポートフォーマット

Media Sortは、ファイルを次の3つのメディアタイプに分類します。

### 1. 画像
サポートされているネイティブ画像形式（純粋なRust製 `image` クレートを使用してデコード）：
- **形式:** JPEG/JPG, PNG, WebP, BMP, TIFF, TGA, Farbfeld (FF), AVIF, DDS, OpenEXR (EXR), HDR, ICO, QOI, および PNM (PBM, PGM, PPM, PAM)。
- **GIFの扱い:** すべてのGIFファイル（アニメーションおよび静止画）は動画として分類され、動画レンダリングスタックへルーティングされます。これにより、グリッド表示やプレビューパネルで自動再生、ループ、一時停止などのコントロールが可能になります。

### 2. 動画＆コンテナ
サポートされている動画形式（システムにリンクされた `libmpv` とその下位のFFmpegデマキサを使用して動的に処理）：
- **基本形式:** MP4, MKV, WebM, AVI, MOV, WMV, FLV, M4V, GIF（ハードコード、常に利用可能）。
- **動的形式:** アプリケーションは起動時に `libmpv` に `demuxer-lavf-list` で追加フォーマットのサポートを問い合わせます。例: MPEG/MPG, TS, VOB, 3GP, OGM。
- **動的依存関係:** コーデックやコンテナ形式の互換性は、ユーザー環境にインストールされている `libmpv` のバージョンおよびその下位にあるFFmpeg（`libavcodec`/`libavformat`）のビルド構成に完全に依存します。`libmpv` がインストールされていない場合でも、アプリは画像と音声に対しては機能しますが、動画再生は無効になります。

### 3. 音声
サポートされているネイティブ音声形式（Symphonia / Rodioを使用してデコード）：
- **形式:** MP3, FLAC, OGG (Vorbis), WAV, AAC, M4A (MPEG-4 Audio / ALAC), WMA, OPUS, AIFF。

## メタデータの抽出

メディアファイルを選択すると、メタデータパネルにファイル構造から直接読み取られた詳細情報が表示されます。

| メディアタイプ | 抽出されるメタデータ属性 | 使用ライブラリ |
| :--- | :--- | :--- |
| **画像** | EXIFフィールド（カメラの機種、露出時間、ISO感度、F値、撮影日時、GPS座標、画像サイズ） | `kamadak-exif` |
| **音声** | オーディオタグ（タイトル、アーティスト、アルバム、リリース年、ジャンル、トラック番号） | `id3` / `metaflac` / `mp4ameta` / `symphonia` |
| **動画** | コンテナタグメタデータ（タイトル、アーティスト、アルバム、年、ジャンル、トラック、総トラック数 — MP4/M4V/MOV/MKV/WebM対応） | `mp4ameta` / `symphonia` |

## 検索フィルタリング

検索バーを使用して、現在のファイルリストを検索およびフィルタリングできます（`I` キーでフォーカスします）。

- **ファイル名検索:** ファイル名の一部を入力するだけで、グリッドを即座に絞り込みます（例: 「vacation」での検索）。
- **拡張子フィルタ:** 拡張子はファイル名の一部であるため、ファイルの拡張子で絞り込みが可能です（例: `.mp4` や `.flac` と入力して該当形式のファイルのみを表示）。

*注意: フィルタリングはファイル名に対して大文字・小文字を区別せずマッチングを行います。ファイル内部のメタデータ（EXIFタグや音声のアーティスト名など）は検索の対象外です。*

## パフォーマンスとキャッシュ

スクロールや移動などの操作を完全にシームレスに保つため、Media Sortには高度なバックグラウンドプリフェッチおよびキャッシュシステムが実装されています。

- **サムネイルのプリフェッチ:** バックグラウンドのワーカータスクを自動的に立ち上げ、現在開いているディレクトリ内の次のファイルのサムネイルを事前に生成します。
- **音声カバーアート:** 埋め込みカバーアートを含む音声ファイル（MP3、FLAC、M4A、OGGなど）は、カバー画像から自動的にサムネイルを生成します。
- **LRUキャッシュ:** サムネイル（最大200個のハンドル）およびフル解像度プレビュー（最大20個のハンドル）を高速なLRU（Least Recently Used）キャッシュに保存し、即時の表示再開を保証します。
- **非同期のファイルシステム監視 (Watcher):** アプリケーションの外部で行われたファイル変更（追加、削除、変更など）を自動的に監視し、UIグリッドをリアルタイムで同期します。
