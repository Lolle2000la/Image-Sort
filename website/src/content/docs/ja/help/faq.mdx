## 一般的な問題

### グリッドにファイルが表示されません

- 対応しているメディアファイルが含まれるフォルダを開いているか確認してください。<Kbd>O</Kbd>を押してフォルダ選択ダイアログを開きます。
- ファイルの拡張子が認識されるものか確認してください。対応フォーマットの全リストは[メディア対応状況](/manuals/media-capabilities/)のページをご覧ください。
- 検索中の場合は、検索バーのクリアボタン（×）をクリックしてフィルターをリセットし、全ファイルを表示してください。

### 動画が再生されません（プレビューが空白またはプレースホルダーが表示されます）

動画再生には**libmpv**がシステムにインストールされている必要があります。

- **Linux:** パッケージマネージャーから`libmpv-dev`または`libmpv1`をインストールしてください（例: Debian/Ubuntuでは`sudo apt install libmpv2`）。
- **macOSおよびWindows:** mpvはアプリケーションに同梱されています。それでも動画が動作しない場合は、GPUドライバが最新でVulkanをサポートしているか確認してください。

libmpvがなくても画像の表示と音声の再生は可能です。**「システムプレーヤーで開く」**ボタンを使用して、デフォルトのメディアプレーヤーで動画ファイルを再生してください。

### アプリケーションが起動しない、または起動時にクラッシュします

- **古い設定:** `config.toml`ファイルを削除してみてください。保存場所はOSによって異なります。[設定ファイル](/advanced/config-file/)のページをご覧ください。次回起動時にデフォルト設定で再作成されます。
- **Vulkan / GPUの問題:** Media Sortはレンダリングにwgpuを使用しており、Vulkan、Metal、またはDirectX 12のサポートが必要です。GPUドライバを更新し、グラフィックススタックが正常に機能していることを確認してください。
- **システムライブラリ不足:** Linuxでは、`libx11`、`libwayland`、`libasound2`がインストールされていることを確認してください。

### すべての設定をデフォルトにリセットするには？

[設定ファイル](/advanced/config-file/)のページに記載されている場所にある`config.toml`ファイルを削除してください。Media Sortは次回起動時に新規の設定ファイルを作成します。

### 古いWPF v2.xバージョンからアップグレードしました。設定はどこにありますか？

Media Sortは初回起動時にレガシーWPFアプリケーションから設定を自動的に移行します。移行が機能しなかった場合は、古い`config.json`ファイルが期待される場所に存在するか確認してください。

- `$CONFIG_DIR/Image Sort/config.json`

ファイルが存在するにもかかわらず設定が移行されなかった場合は、新しい`config.toml`を削除してアプリケーションを再起動してください。移行は`config.toml`が存在しない場合にのみ1回だけ実行されます。

### Media Sortを完全にキーボードだけで操作できますか？

はい。Media Sortはキーボードファーストの操作を前提に設計されています。フォルダの移動、ファイルの選択、移動/コピー/削除、名前変更、フォルダ作成、ピン留めフォルダの管理。すべての操作にキーボードショートカットがあります。完全なリファレンスは[キーボードレイアウト](/manuals/keyboard-layout/)のページをご覧ください。

### 削除したファイルはどこにありますか？

削除されたファイルはOSのゴミ箱（Windowsではごみ箱、macOS/Linuxではゴミ箱）に移動されます。完全に削除されるわけではありません。Media Sortの実行中は、<Kbd>Q</Kbd>を押して削除を取り消し、ファイルを元の場所に復元できます。アプリを閉じた後も、システムのゴミ箱から手動でファイルを復元できます。

### Media Sortは元のファイルを変更しますか？

仕分け操作（移動、コピー、名前変更、削除）はファイルの場所や名前を変更しますが、Media Sortがファイルの**内容**を変更することは決してありません。メタデータ抽出は読み取り専用です。画像や動画ファイルが再エンコードされたり変更されたりすることはありません。
